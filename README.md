## ğŸ“Œ Project Overview

This project is a Python-based **ETL (Extract, Transform, Load)** script designed to clean messy, real-world customer data.

In data analytics, "dirty data" (inconsistent formats, duplicates, missing values) is a common bottleneck. This project simulates a scenario where a client sends a weekly dump of raw customer records containing various errors. The script automates the cleaning process to produce a standardized, analysis-ready Excel file.

## ğŸ’¼ The Scenario

**The Problem:** A client provides a weekly `csv` dump where:

* **Dates** are in mixed formats (e.g., `DD-MM-YYYY`, `MM/DD/YYYY`, and `YYYY.MM.DD`).
* **Phone numbers** contain special characters (parentheses, dots, dashes).
* **Emails** are sometimes missing or { invalid (e.g., missing top-level domains like `.com`). } **HomeWork**
* **Duplicates** and **Null values** are scattered throughout.

**The Solution:** A Python script that ingests the raw CSV, applies validation logic, and exports a polished Excel report.

## âš™ï¸ Key Features

* **Date Standardization:** Converts mixed date formats into a standard ISO format (`YYYY-MM-DD`).
* **Regex Pattern Matching:**
* **Phone:** Strips non-numeric characters to standardize format.
* **Email:** Validates structure to flag invalid entries (e.g., `david@example` is flagged/removed). **HomeWork**


* **Duplicate Removal:** Identifies and drops duplicate records based on ID and Name.
* **Null Handling:** Intelligently fills missing values (e.g., "Unknown" for names), ("No Email" for emails) **HomeWork**.
* **Dynamic Export:** Saves the final output with a timestamped filename (e.g., `cleaned_data_2023-10-25.xlsx`).
